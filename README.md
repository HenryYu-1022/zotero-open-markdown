# Zotero Open Markdown

中文说明见 [README.zh-CN.md](README.zh-CN.md)。

---

A lightweight [Zotero 7](https://www.zotero.org/) plugin that adds right-click context menu options to **reveal or open** the corresponding Markdown file for any Zotero item — bridging your Zotero library with your [paper-agent](https://github.com/HenryYu-1022/paper-agent) Markdown corpus.

## Why This Plugin Exists

If you use [paper-agent](https://github.com/HenryYu-1022/paper-agent) to convert your Zotero PDF library into Markdown, you have a rich, searchable text corpus on disk. But when you're working in Zotero and want to quickly jump to a paper's Markdown file — to read it in Obsidian, feed it to an AI agent, or just browse the converted text — there's no built-in way to do it.

This plugin adds two simple right-click menu items:

- **Reveal Markdown in Finder** — highlights the Markdown file/folder in your OS file manager (Finder on macOS, Explorer on Windows)
- **Open Markdown File** — opens the `.md` file with your system's default Markdown editor (e.g., Obsidian, Typora, VS Code)

## How It Works

```text
Zotero Item → PDF Attachment → input_root/<path>/Paper.pdf
                                        ↓ (file name mapping)
                               markdown_root/<path>/Paper/Paper.md
                                        ↓
                               Reveal in Finder / Open with editor
```

The plugin reads the PDF attachment's file path, computes the relative path from your configured `input_root`, then looks for the corresponding Markdown bundle folder under `markdown_root`. This matches the directory convention used by [paper-agent](https://github.com/HenryYu-1022/paper-agent).

## Installation

### Requirements

- **Zotero 7** or later
- A Markdown library generated by [paper-agent](https://github.com/HenryYu-1022/paper-agent) (or any tool that follows the same `<stem>/<stem>.md` folder convention)

### Install from .xpi

1. Download the latest `zotero-open-markdown.xpi` from the [Releases](https://github.com/HenryYu-1022/zotero-open-markdown/releases) page
2. In Zotero, go to **Tools → Add-ons**
3. Click the gear icon ⚙ → **Install Add-on From File…**
4. Select the downloaded `.xpi` file
5. Restart Zotero if prompted

### Build from source

```bash
git clone https://github.com/HenryYu-1022/zotero-open-markdown.git
cd zotero-open-markdown
bash scripts/build.sh
```

This creates `zotero-open-markdown.xpi` in the project root. Install it using the steps above.

## Configuration

After installation, configure the plugin in **Zotero Preferences → Open Markdown**:

| Setting | Description | Example |
|---------|-------------|---------|
| **Markdown Library Root** | The root directory of your converted Markdown files. This is your paper-agent `output_root/markdown/` directory. | `/Users/you/Documents/paper-library/output/markdown` |
| **PDF Library Root** | The root directory of your source PDFs. This should match the `input_root` in your paper-agent `settings.json`. | `/Users/you/Documents/paper-library/input` |

> **Tip:** If you use [zotero-attanger](https://github.com/HenryYu-1022/zotero-attanger) to sync PDFs to Google Drive, the PDF Library Root should point to the attanger export directory.

## Usage

1. Select any item in Zotero that has a PDF attachment
2. Right-click to open the context menu
3. You will see two new options at the bottom:
   - **Reveal Markdown in Finder** — opens the Markdown file's location in your file manager
   - **Open Markdown File** — opens the `.md` file with your default editor

If no corresponding Markdown file is found, a helpful error message will appear with troubleshooting guidance.
